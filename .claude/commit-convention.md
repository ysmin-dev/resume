# 커밋 메시지 컨벤션

커밋 메시지는 아래 타입 접두사를 사용한다: `<type>: <설명>`

| 타입 | 설명 |
| --- | --- |
| fix: | 버그 수정 |
| docs: | 문서 수정 (docs/, README.md 등) |
| style: | 코드 포맷팅, 세미콜론 누락 등 (비즈니스 로직 변경 없음) |
| refactor: | 코드 리팩토링 (기능 변화 없이 구조만 개선) |
| test: | 테스트 코드 추가, 수정 |
| chore: | 빌드, 배포 스크립트, 의존성(package.json 등) 설정 등 |
| perf: | 성능 개선 |
